Data Quality Information

 Section Index

    Logical Consistency Report:
      The logical consistency of the WALLS station shapefiles is given by the unit variance estimate (UVE). The UVE is a consistency measure closely analogous to sample variance. It should correlate with expected survey accuracy when there is a sufficient number of surveyed loops. Smaller UVEs are better. The best cave surveys typically produce UVEs with values less than 2.0. The loop counts measure the significance of UVEs and allow those from different data sets to be correctly averaged: UVE_total = (LOOPS1*UVE1+LOOPS2*UVE2)/(LOOPS1+LOOPS2).
    Completeness Report:
      Incomplete. This data set does not contain data for all known, or all surveyed, passages between the Historic and Violet City Entrances. It consists of 4,273 points describing 38,270.2 meters of surveyed passages between the Historic and Violet City Entrances.
    Positional Accuracy:
      Horizontal Positional Accuracy:
        Horizontal Positional Accuracy Report:
          When weighting compass and tape traverses for least squares adjustment, the default error variance assumed (by Walls) for each horizontal component is <total traverse length in feet>/100 ft², which is equivalent to a standard deviation of 0.3 m for a 30-meter vector measurement. This being only a rough approximation of variance, the component errors are currently treated as if they were statistically similar and uncorrelated. Under these assumptions the expected horizontal unit variance estimate (UVE) resulting from an adjustment is 1.0. Cave surveys, however, are significantly affected not only by random errors in instrument and tape readings, but also by station positioning errors and systematic errors due to imperfect instrument calibration (among other things). Cave difficulty and skill of surveyors are important factors. When there are sufficient surveyed loops, the effects of these errors normally result in UVEs ranging from 1.0 to 2.0 in surveys believed to be of good quality. (CRF surveying methods and standards are assumed.)
          The accuracy of a station's estimated UTM coordinates will depend on its distance from well-established benchmarks and how much control is provided by surveyed loops. Approximate confidence intervals can be computed from statistics derived from the least squares adjustment and error analysis. The required statistics are provided in the following numeric fields of the station shapefile:

          SD_H - A priori standard deviation of each horizontal component's error in meters.
          SDUW_H - A posteriori standard deviation of unit weight for each horizontal component.
          LOOPS_H - Loop count (degrees of freedom) for each horizontal component.

          (Note that the standard deviation of unit weight is the square root of the UVE.) These numbers can be used to derive any of several popular kinds of approximate confidence intervals. For example, an a priori bound on either the Easting or Northing of a UTM position, at 90% confidence, would be computed as plus or minus 1.64 x SD_H. The radius of an approximate 90% confidence region would be 2.15 x SD_H. These intervals depend only on network geometry and the default variances used for weighting measurements. To obtain the corresponding a posteriori confidence interval, which depends on observed consistency (loop closures), we would replace SD_H with SDUW_H x SD_H. This will be a legitimate confidence interval only if the loop count is large enough, say over 20 (generally the case for CRF data submissions to MCNP). For fewer loops, the radius of a confidence region can be computed using critical values of the F distribution, with 2 and 2 x LOOPS_H degrees of freedom. If there are no loops, only a priori bounds are possible.

      Process Step:
        Process Description:
          The unit variance estimate (UVE) is a computed byproduct of a least-squares adjustment, and should have a value close to one when the number of loops in a network is sufficiently large and the error variances used to assign weights to measurements are realistic. The default variances assigned by Walls result in UVEs ranging from 0.2 to 2 for typical "good" cave surveys. Very large values indicate abnormal inconsistency in the data. Walls breaks the UVE into two components -- horizontal (UveH) and vertical (UveV) -- and displays them for each loop system.
          When Walls does a least-squares adjustment, it assumes that the error variances of the observed X,Y, and Z components of a traverse are each proportional to the traverse's total length. By default, Walls converts a compass and tape survey vector to its east-north-up components and assigns to each component a variance of <total length in feet>/100 ft². This is equivalent to a standard deviation of 0.3 m for a 30-meter shot. The variance of a traverse is the sum of the variances of its component vectors.

          The result of the adjustment is a final estimated displacement for each traverse: x, y, z, and the theoretical variances of this estimate (used for error analysis). Also obtained are the loop counts, Nh and Nz, and the quantities actually minimized, the sums of squares of weighted residuals:

          SSh = Sum over all unconstrained traverses of ((X - x)² + (Y - y)²) / Vh, and
          SSz = Sum over all unconstrained traverses of (Z - z)² / Vz

          The unit variance estimates are then computed as

          UVEh = SSh / (2·Nh) and UVEz = SSz / Nz.

          If it is assumed that the assigned variances are correct apart from an unknown scaling factor (i.e., the unit variance), the UVE can be interpreted as a sample variance that estimates this scaling factor. Its expected value after data screening could be significantly different from one depending on the survey's quality (grade), skill of surveyors, and difficulty.

      Attribute:
        Attribute Label: NAME
        Attribute Definition:
          Station name. Usual format for Cave Research Foundation surveys at Mammoth Cave National Park: <FSB No.><Survey letter><Number>(Example: 196A200) Other formats are technically possible. Except in special cases (e.g., Walker benchmarks like TT8W), the MCNP station names should conform to the CRF naming convention (<FSB No.><Survey letter><Number>). In Walls projects, a name can also have a prefix qualifier, delimited by a colon, to ensure uniqueness across a project with multiple caves or sub-projects. (Example: HISTORIC:TT8W.) Unprefixed names are limited to 8 characters in length while prefixes technically can be up to 128 characters long. Having concluded that prefixes of 8 characters or less will be sufficient if needed at all, we have chosen a 17-character field length for station names in CFF's MCNP shapefiles.

    Overview Description:
      Entity and Attribute Overview:
        WALLS station shapefiles define nodes (survey stations) in the cave survey data network. Each station record is assigned attributes signifying its name, X coordinate, Y coordinate, and Z coordinate. In addition, each station record also has the following attributes when data is available: distance to left wall of cave passage, distance to right wall, distance to ceiling, distance to floor, azimuth of line perpendicular to plane defined by these distances, and name of CRF Cartographic Program map sheet taken from the Index of Map Sheets.
